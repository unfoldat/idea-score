어떻게 써야 검색이 내 글을 답으로 선택할까?

AEO 스킬 — 질문을 담는 정보 구조 설계  ·  JSON-LD 스키마  ·  검색·AI 최적화

SEO vs AEO — 뭐가 다른가?

SEO (검색엔진 최적화)
목표: 검색 목록에서 클릭 유도
키워드 밀도, 백링크, 페이지 속도
결과: 사람이 링크를 클릭해야 도달
아직 유효 — 브랜드·탐색 쿼리
AEO (답변 엔진 최적화)
목표: AI·검색이 직접 내 글을 인용
구조화 데이터, 질문 중심 제목
결과: 클릭 없이도 노출 가능
정보형·방법형 쿼리에서 결정적

왜 구조가 중요한가? — P≠NP로 보기

검색엔진 = certificate 검증자

구조 있음 (P)
JSON-LD 스키마 = certificate
검색엔진이 즉시 검증 가능
→ 빠른 featured snippet 선택
구조 없음 (NP-hard)
기계가 의미를 추론해야 함
오해·누락 발생
→ 경쟁 페이지에 밀림
"내 글이 이런 질문에 대한 답입니다"를 기계가 읽을 수 있게 선언하는 것 = certificate 제시.
출처를 명시하는 것이 정보 신뢰도를 높이듯, 스키마는 콘텐츠 의미를 검색엔진에 확인시킨다.

FAQPage 스키마 — 최소 예시

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "FAQPage", "mainEntity": [ { "@type": "Question", "name": "질문을 여기에 씁니다?", "acceptedAnswer": { "@type": "Answer", "text": "답변을 여기에. 200자 이내 권장." } } ] } </script>

AEO 글쓰기 체크리스트 — 5단계

  1. 01
    제목을 질문 형태로 쓴다
    "방법" → "어떻게 하나요?", "개념" → "이게 뭔가요?"   검색자의 실제 입력 쿼리와 가장 가까운 형태가 매칭에 유리합니다.
  2. 02
    첫 문단에서 한 문장으로 답한다
    검색엔진은 첫 150~200자를 featured snippet 후보로 봅니다. 서론을 길게 쓰기 전에 핵심 답변 한 문장을 먼저.
  3. 03
    소제목도 질문 형태 (H2/H3)
    각 섹션 제목이 독립적인 검색 쿼리가 될 수 있어야 합니다. "배경" → "왜 이렇게 됐나요?", "결론" → "결국 무엇을 해야 하나요?"
  4. 04
    JSON-LD 스키마 추가
    FAQPage · HowTo · Article 중 적합한 타입 선택. <head>에 삽입. Google Rich Results Test로 검증.
  5. 05
    canonical URL 선언
    <link rel="canonical" href="https://..."> 중복 콘텐츠 방지 + 링크 권위 집중. 반드시 <head>에 포함.

누가 검색하는가? — 전환점(transition point) 타겟

정적 페르소나 (약함)
"30대 직장인 김씨"
고정된 속성으로 사람을 분류
실제 검색 의도와 괴리 큼
전환점 타겟 (강함)
"처음으로 GitHub Pages 쓰려는 순간"
사람이 아닌 상태(state)를 타겟
그 순간의 질문 = 검색 쿼리 그 자체

전환점에 있는 사람은 이미 질문을 갖고 있습니다. 그 질문에 정확히 답하는 글이 AEO의 핵심입니다.

Idea Score와 AEO 파이프라인

질문 히스토리 → 스킬카드 → 검색 노출

질문 기록
Idea Score → 패턴 발견
반복 질문 → 스킬카드 작성
AEO 구조 → 검색 노출
FAQ 블록
내가 Idea Score에서 기록한 질문들은 내 관심사와 세계관의 투영입니다.
같은 전환점에 있는 사람들이 같은 질문을 검색합니다.
→ 스킬카드 = 내 질문 히스토리로부터 파생된 AEO 콘텐츠

자주 묻는 질문

AEO가 뭔가요? SEO랑 다른가요?
SEO는 클릭을 유도하고, AEO는 검색·AI가 직접 내 글을 답으로 인용되게 합니다. 정보형 쿼리("어떻게 하나요?", "~가 뭔가요?")에서 AEO가 점점 더 중요해지고 있습니다. 구조화 데이터(JSON-LD)와 질문 중심 글쓰기가 핵심입니다.
JSON-LD를 어디에 어떻게 넣나요?
<head> 태그 안에 <script type="application/ld+json">으로 삽입합니다. HTML 본문 구조와는 별개로 기계용 메타데이터입니다. Google Rich Results Test(search.google.com/test/rich-results)로 오류 확인 가능합니다.
독자를 정확히 특정해야 하나요?
정적 페르소나보다 전환점(transition point)으로 생각하세요. "어떤 사람"이 아니라 "어떤 상태에 있는 순간"을 타겟하면 다양한 사람이 같은 질문을 할 때 내 글이 답이 됩니다.
canonical이 없으면 어떻게 되나요?
같은 콘텐츠가 여러 URL에 있을 때 검색엔진이 어느 것을 원본으로 볼지 결정을 못합니다. 링크 권위가 분산되고 색인 순위가 낮아집니다. 정적 HTML 사이트는 반드시 canonical을 <head>에 포함하세요.
Idea Score와 AEO가 어떻게 연결되나요?
Idea Score에서 기록한 질문 히스토리는 사람들이 실제로 검색하는 질문 패턴과 일치합니다. 각 스킬카드는 그 질문에 대한 AEO 최적화 답변 페이지입니다. 질문 → 스킬 → 스킬카드(AEO)가 하나의 파이프라인이 됩니다.
